Analisis dan Penjelasan

Halaman Daftar dan Login

Task 1 Daftar : Pada akhir halaman daftar, tombol Daftar tidak dapat di akes sehingga tidak bisa berpindah ke halaman selanjutnya. Pada bagian inilah yan kadang – kadang membuat user bingung. Tombol daftar sendiri belum dapat di implementasikan. Untuk kolom password di halaman Daftar dan login user bertanya maksimal karakter untuk password yang diinputkan.

Halaman Lupa Password

Task 2 Lupa Password : Sama halnya seperti pada halaman daftar, tombol Kirim pad lupa password belum diimplementasikan. Sehingga memperbanyak waktu yang dibutuhkan user untuk mengevaluasi halaman tersebut.

Halaman Konfirmasi Pesanan

Task 5 Pesan Makanan / Minuman sampai ada driver : Tidak berbeda dari halaman daftar dan lupa password, pada halaman ini juga Tombol tambah pesanan belum dapat diimplementasi sehingga jika user ingin melihat kembali menu harus menggunakan link Back pada ujung kiri tampilan yang membuat waktu evaluasi semakin lama. Pada task ini juga user dapat berulang kali memesan ataupun kembali ke task selanjutnya yaitu melihat menu, sehingga sulit untuk membuat prediksi waktunya.

Implikasi Hasil Terhadap Rancangan

Pada halaman daftar, login dan halaman lupa password secara umum tidak terdapat komentar yang dapat mengubah requirement rancangan. Rata – rata user yang mengevaluasi berkata rancangan sudah cukup baik dan dapat mengimplementasikan aplikasi yang diinginkan.

Untuk halaman Konfirmasi Pesanan bergantung pada gender, untuk laki – laki biasanya proses pemesanan lebih singkat dibanding user wanita.

Kritik Terhadap Rencana Evaluasi

Sesuai dengan evaluation plan yang sudah di jelaskan pada LH 2a terdapat beberapa aspek yang akan di kaji yaitu

  • Warna : tidak terdapat komentar untuk warna aplikasi. Karena rata – rata user berusia 19 – 20 tahun user lebih menyukai warna yang terang.
  • Waktu : Prediksi waktu dan real waktu evaluasi banyak yang melebihi prediksi dikarenakan banyak hal yang tidak terduga ketika observasi langsung.
  • Fitur : Fitur sudah sama dengan yang diinginkan pembeli namun ada beberapa fitur yang belum terimplementasi seperti yang sudah dijelaskan diatas.
  • Proses pengerjaan Task : sudah sesuai dengan referensi dan tidak membingungkan user.
  • Tampilan : sudah sesuai dengan yang diinginkan user. Tidak terlalu ramai dan tidak terlalu simple.
