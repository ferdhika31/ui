Laporan Hasil 2B

Prototype, 30 Mei 2017

Pembeli Driver

Keterangan Prototype

  1. Pembeli

    2. Tampilan Keterangan

    Masuk ke akun atau daftar akun
    • Text area untuk kolom no HP dan password dapat diisi ataupun tidak, namun button masuk tetap bisa di jalankan.
    • Tombol masuk akan mengarahkan pembeli ke halaman berikutnya.
    • Link lupa password akan mengarahkan pembeli pada page Lupa Password yang berisi text area untuk input email.
    • Tombol Daftar akan mengarahkan pembeli ke halaman pendaftaran akun.

    Menu Utama

    • Kolom pencarian dapat digunakan dengan mengetikkan nama warung ataupun nama menu.
    • Setiap kategori makanan / minuman padamenu utama dapat diakses.

    Kategori terlaris, menu hemat, Daftar warung, Aneka Nasi, Snack dan Jajanan, Minuman, Roti

    • Setiap kategori dapat diakses dengan pindah kehalaman terkait.
    • Di setiap kategori terdapat jenis – jenis menu. Dan jika diklik akan berpindah halaman ke menu lain yang didalamnya terdapat informasi dari menu tersebut.
    • Pada setiap halaman ini juga terdapat tombol “Tambah” yang dapat diakses dan berpindah halaman ke Halaman pemesanan.

    Halaman pemesanan (order)
    • Terdapat area Pesanan Anda, dimana terdapat “Number Stepper” yang fungsinya untuk menambah kuantitas pesanan.
    • Button tambah pesanan belum bisa di akses
    • Kolom Alamat pengantaran dapat diubah dengan mengklik tulisan ubah yang akan berpindah ke halaman ubah tujuan pengantaran.
    • Detail Pembayaran, radio button dapat di jalankan ( dapat memilih pembayaran secara tunai / online )
    • Tombol pesan akan mengarahkan pembeli kehalaman pengantaran yang berisi identitas driver, dan informasi pemesanan.

    Rincian Pesanan

    • Jika pesanan sudah diterima, maka pembeli dapat mengklik tombol “Pesanan telah sampai” dan melakukan konfirmasi.

    Beri ratting pada driver
    • Tombol selesai dan bintang untuk ratting dapat diakses.

    Ikon keranjang
    • Dapat diakses yang fungsinya dalah untuk melihat review pesanan, baik yang sudah selesai maupun yang sedang diproses.

    Ikon orang

    (biasanya digunakan untuk akun)
    • Dapat diakses yang halamannya akan beralih ke akun milik pembeli.
    • Pada halaman pembeli, belum dapat mengubah informasi dari akun pemebli

  2. Driver

    3. Tampilan Keterangan

    Masuk Akun
    • Text area untuk kolom no HP dan password dapat diisi ataupun tidak, namun button masuk tetap bisa di jalankan.
    • Tombol masuk akan mengarahkan driver ke halaman berikutnya.

    Menu Utama
    • Daftar Order terdiri dari daftar order yang dapat dipilih dengan menekan button Lihat Detail, dan akan mengarahkan user ke Tampilan Konfirmasi Pemesanan
    • Icon on/off akan merubah warna button tersebut(hijau ON, putih OFF)
    • Icon Keranjang akan mengarahkan user ke tampilan Riwayat Order
    • Icon Man akan mengantarkan user ke tampilan Profil

    Komfirmasi Pemesanan
    • Button Back akan mengantarkan user menuju tampilan Menu Utama
    • Button Lihat Catatan akan menampilkan popup berupa catatan dari Alamat Antar
    • Button Ambil akan menampilkan popup berupa Konfirmasi Tunai, lalu user dapat menekan button OK untuk menuju tampilan Order

    Order
    • Button Selesai berfungsi sebagai proses pengerjaan telah selesai, dan akan menampilkan popup berupa Konfirmasi Kembali, lalu user dapat menekan button OK akan mengantarkan user menuju tampilan Riwayat Order

    Riwayat Order
    • Dengan menekan salah satu dari list Riwayat Order maka user akan diantarkan menuju halaman Detail Order yang sudah diselesaikan
    • Button FoodJas akan mengantarkan user menuju tampilan Menu Utama

    Detail Order
    • Button Back akan mengantarkan user menuju tampilan Riwayat Order

    Tampilan Akun
    • Button Keluar akan mengantarkan user menuju tampilan Masuk Akun

Kesimpulan :

Prototype dibuat menggunakan tools IONIC yaitu sebuah framework yang digunakan untuk membangun aplikasi mobile hybrid dengan HTML5, CSS dan AngularJS. Pada lh 2b ini kami membuat prototype yang dapat di demokan langsung di web LH kami sehingga kami menggunakan framework IONIC.

90% dari tampilan yang dibuat di mockup baik itu untuk pembeli maupun driver dapat dibuat prototype nya. Hanya saja ada beberapa prototype yang belum terealisasikan seperti :

  • Tambah pesanan pada tampilan pembeli
  • Telpon Pemesan pada tampilan driver

Dan untuk tampilan sendiri terdapat beberapa revisi, seperti bahasa yang tidak konsisten. Sebelumnya bahasa yang terdapat pada tampilan ada yang menggunakan bahasa inggris dan bahasa indonesia. Pada prototype sudah di rubah kedalam bahasa indonesia. Pada setelan akun terdapat opsi ubah bahasa, namun belum bisa direalisasikan.
